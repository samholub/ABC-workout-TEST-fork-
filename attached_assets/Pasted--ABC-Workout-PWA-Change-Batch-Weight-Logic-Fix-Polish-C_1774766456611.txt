### 1. CRITICAL BUG FIX: Working Weight Uses Mode Instead of Max

**Problem:** `getProgression()` and `getSuggestedLoad()` use `Math.max` across all sets to determine working weight. If a user accidentally loads too heavy on one set out of three, the app treats that outlier as the working weight and suggests progressing from it.

**What to do:**

A) Add this helper function in Section 5, BEFORE `getProgression`:

```javascript
function getWorkingWeight(sets){
  var weights=sets.filter(function(s){return s.weight>0}).map(function(s){return s.weight});
  if(weights.length===0)return 0;
  var freq={};weights.forEach(function(w){freq[w]=(freq[w]||0)+1});
  var modeW=0,modeCount=0;
  Object.keys(freq).forEach(function(w){if(freq[w]>modeCount){modeCount=freq[w];modeW=Number(w)}});
  if(modeCount>1)return modeW;
  weights.sort(function(a,b){return a-b});
  var mid=Math.floor(weights.length/2);
  return weights.length%2===0?weights[mid-1]:weights[mid];
}
```

B) In `getProgression`, find this line:
```javascript
var maxW=Math.max.apply(null,sets.map(function(s){return s.weight||0}));
```
Replace with:
```javascript
var maxW=getWorkingWeight(sets);
```

C) In `getSuggestedLoad`, find this line:
```javascript
var w=Math.max.apply(null,ex.sets.map(function(s){return s.weight||0}));
```
Replace with:
```javascript
var w=getWorkingWeight(ex.sets);
```

**Verify:** If sets are logged at 35, 35, 50 lbs, working weight resolves to 35. Progression suggests 40 (35+5), not 55.
